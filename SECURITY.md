# 보안 정책

## 지원 범위

보안 수정은 기본 브랜치인 `main`의 최신 버전을 대상으로 합니다. 과거 커밋이나 별도 배포본은 별도로 지원하지 않을 수 있습니다.

## 취약점 신고

취약점은 공개 Issue, Discussion, PR 또는 커밋 메시지에 작성하지 마세요.

가능하면 GitHub 저장소의 **Security → Report a vulnerability** 기능을 사용해 비공개로 신고해 주세요.

- 비공개 신고: [GitHub Security Advisory 작성](https://github.com/xericen/GACHI/security/advisories/new)
- 비공개 신고 기능이 보이지 않으면, 공개 이슈에는 상세 내용을 적지 말고 안전한 연락 방법만 문의해 주세요.

신고에는 가능한 범위에서 다음 내용을 포함해 주세요.

- 영향받는 기능과 버전 또는 커밋
- 재현 조건과 최소 재현 절차
- 예상 영향과 악용 가능성
- 로그·화면·요청 예시에서 개인정보와 비밀값을 제거한 증거
- 알고 있다면 완화 또는 수정 제안

## 처리 절차

1. 신고 접수 사실을 확인합니다.
2. 영향 범위와 심각도를 검토합니다.
3. 비공개 브랜치에서 수정하고 회귀 검증을 수행합니다.
4. 필요한 키 폐기·재발급과 운영 완화 조치를 진행합니다.
5. 수정 배포 후 공개 범위와 시점을 신고자와 조율합니다.

## 민감 데이터

다음 정보는 저장소, 이슈, PR, Actions 로그에 포함하면 안 됩니다.

- API 키, 액세스 토큰, JWT 비밀값과 개인키
- DB 접속정보, 운영 서버 자격증명과 서비스 계정 파일
- PASS 본인 인증 원문 결과
- 사용자의 위치, 채팅, 연락처와 기타 개인정보
- 운영 DB 덤프, 업로드 파일과 내부 로그

노출이 의심되면 Git에서 파일을 삭제하는 것만으로는 충분하지 않습니다. 해당 자격증명을 즉시 폐기·재발급하고, 원격 Git 이력과 Actions 로그의 노출 여부도 확인해야 합니다.
